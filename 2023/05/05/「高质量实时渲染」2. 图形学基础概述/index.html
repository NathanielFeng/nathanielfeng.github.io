「GAMES202」图形学基础概述

1. GPU 渲染管线

对于渲染管线来说，一开始的输入是 3D 的模型，而最后的输出是一张图片。

首先，各种各样的 3D 物体在空间中是由一些点 和 点与点之间的连接关系 组成的。

任何点都会经过第一个步骤：顶点的处理。

顶点的处理主要是一系列的变换（如：MVP变换 ，视口变换），将世界坐标系下的点转到屏幕坐标系下。

我们知道，顶点在经过这一系列的变换之后，它们的连接关系是没有改变的。

所以之前的一个三角形，原来是哪三个顶点，现在还是这三个顶点，只不过被投影到屏幕上去了。

既然三角形被投影到屏幕上去了，下一步我们要做的是什么呢？光栅化！

光栅化是在干什么？简单来说，就是要把原本一个连续表示的三角形，把它离散成屏幕上的一堆像素(pixel/fragment)。

离散成像素的过程中，肯定会涉及到遮挡的处理：哪些三角形在前面，哪些三角形在后面。

我们是通过深度缓存记录每个像素的最小的深度值来处理的。

最后，当确定了可见的 fragment 之后，就可以对 fragment 做着色，着色就是去计算 fragment 的颜色值。

2. OpenGL

OpenGL 是什么？

—— OpenGL是一系列的 API，是在 CPU 端上执行的，负责调度 GPU 运作。

OpenGL 的优缺点：

  • 优点：跨平台
  • 缺点：版本太零碎、C 风格代码，不容易学

如何去理解 OpenGL：

一种重要的类比：画油画！

  • A：放置物体 / 模型
  • B：设置画架的位置
  • C：把画布放在画架上
  • D：在画布上画画
  • E：(如果还想画更多的话，将其他画布固定在画架上并继续绘画)
  • F：(之前画过的画可以当作参考拿过来用)

A. 放置物体 / 模型

  1. 告诉程序是什么模型

    OpenGL 定义了一个 Vertex Buffer Object(VBO)，它是 GPU 中的一块区域，用于存储模型的顶点，法线，纹理坐标等，它与 .obj 文件非常类似。

  2. 摆放模型的位置

    使用 OpenGL 函数直接获取矩阵，如：glTranslate, glMultMatrix...完全无需自己手写矩阵。

B. 设置画架的位置

  1. 视图变换

    简单地通过调用 gluPerspective 就可以设置相机（是透视投影 / 正交投影等等）

  2. 创建一个 framebuffer(OpenGL中的画架)

C. 把画布放在画架上

​ 只要指定了一个 framebuffer，是可以一下渲染出很多不同的纹理。

​ 即在 OpenGL 中一次渲染（即一个pass），一个 pass可以输出多张图到一个 framebuffer。

​ （垂直同步、双重缓冲、三重缓冲）

D. 在画布上画画

  1. 如何去执行 shading（vertex shader / fragment shader）

    OpenGL 调用用户定义的 vertex shader 来操作顶点的变换

    OpenGL 调用用户定义的 fragment shader 进行像素的着色

总结：在每个 pass 中

  • 指定物体、相机、MVP等

  • 指定 framebuffer 和 输入输出纹理

  • 指定 vertex / fragment shader

  • 开始渲染！

  • 多次 passes！（使用之前渲染得到的纹理作为这次的参考，比如 shadow map）

GLSL是什么？

—— GLSL 是用来描述着色器如何运作的一种语言。

Shader 的设置

4. 渲染方程

渲染中最重要的方程，描述了光线是如何传播的： \[ L_o(p, \omega_o) = L_e(p, \omega_o) + \int_{H^2}f_r(p, \omega_i \rightarrow \omega_o) \ L_i(p, \omega_i) \ cos\theta_i \ d\omega_i \] 在实时渲染中，可见性会被显式考虑： \[ L_o(p, \omega_o) = \int_{\Omega+}L_i(p, \omega_i) \ f_r(p, \omega_i, \omega_o)cos\theta_i \ V(p,\omega_i) \ d\omega_i \] 这样的好处的是，环境光照的理解会更为直观。

如何解决方程的递归？

​ 全局光照 = 直接光照 + 一次的间接光照（一次和更多次的间接光照区别不大）
