「高质量实时渲染」实时阴影

1. Shadow Mapping

Shadow Mapping 是什么？Shadow Mapping 是一个 2-Pass 的针对点光源的硬阴影算法。

要渲染一个点光源在场景中投射出的阴影就得用 Shadow Mapping 的技术，它是一个两趟的算法，也就是说，它会渲染场景两遍。

第一遍渲染场景的时候 ，会从 Light 所在的地方看向这个场景，然后渲染场景的时候输出一个 Light 所能看到的最近的深度，这就是第一个 pass；

图1  从光源输出一个“depth texture”

第二遍渲染场景的时候，会从相机位置出发渲染一遍整个场景，在这个过程中配合 Light pass 得到的 Shadow Map，就可以去检测相机看到的任何一个点是否在阴影里，或者是否被 Light 所看到；

图2  从眼睛出发渲染一个正常的图像

关于深度需要注意的一点

在做深度比较的时候，很重要的一点是保证 2 趟 pass 之间深度的一致。为什么会这么说？这是因为在做透视投影的时候，物体的 z 值会因为投影而发生变换（会被推向远平面）。所以要么 2 趟 pass 都用的是投影之后的 z 值；要么 2 趟 pass 都用的是实际的距离。

Shadow Mapping 是一个图像空间的算法

优点：只需要知道 Shadow Map 就可以了，不需要知道场景中几何信息。

缺点：会出现自遮挡与走样问题 。

2. Shadow Mapping 的缺点

自遮挡问题

Shadow Mapping 的第一个 Pass 记录的从 Light 出发看到的最近的深度会放到深度图 Shadow Map 中，其中每一个像素的值都代表一个深度值。

问题来了，由于每个像素都对应场景中的一块连续的区域，根据深度图，这块区域的深度都是一样的，但是实际上这块区域的深度值并不一样，有大有小，这样就会造成实际的深度值与记录的不一样！

这种情况在 Light 垂直于地板的时候问题最小，在 Light 平行于地板的时候问题最大。

图3  左：RTR4中的图片，右：自遮挡问题示意图

比如上面左图的地板便是由于自遮挡造成的问题，我们可以根据右图来理解。

Shadow Map 中的每个像素看到的地板并不是图中黑线，而是红线！这是因为实际的深度是连续的，而深度图中的深度由于像素的存在是不连续的，是个常数！所以才造成每个像素眼中的地板都是斜的！

如何解决自遮挡问题？

通过添加一个 偏差 Bias（根据角度自动调整） 来减少自遮挡。

即就算有物体在 Bias 范围内，也不算物体遮挡了光源看见的点。但这样也会造成问题：

图4  左：detached shadow问题，右：bias范围内的物体不算遮挡

如何解决detached shadow问题？（detached：单独的、脱离的）

—— Second-depth shadow mapping

每次记录深度的时候，不仅存最小的深度，还要存第二小的深度，然后比较深度的时候，用最小深度和第二小深度中间的深度来做比较。

图5  second-depth shadow mapping

但是实际中没有人用，这是因为该算法对物体有要求，它要求物体必须的 watertight 形状的（即不能是一张纸这样，物体有正面也必须有反面）；而且计算开销大！

走样问题

图6  shadow mapping中的走样问题

3. Shadow Mapping背后的数学

在实时渲染中，我们更关心“近似相等”，其中一个非常重要的不等式是： \[ \int_\Omega f(x)g(x)dx \approx \dfrac{\int_\Omega f(x)dx}{\int_\Omega dx} \cdot \int_\Omega g(x)dx \] 上面式子右边第一项相当于 \(f(x)\) 在 \(\Omega\) 上的均值，其分母是归一化的常数。

那么什么时候做是准确的？满足下面两个条件之一即可：

  • 当实际的积分范围很小的时候；

  • 当 \(g(x)\) 足够光滑（换句话说 \(g(x)\) 在定义域内差别不大，低频）；

当对渲染方程使用上面的不等式之后： \[ L_o(p, \omega_o) = \int_{\Omega+}L_i(p, \omega_i) \ f_r(p, \omega_i, \omega_o) \ cos\theta_i \ V(p,\omega_i) \ d\omega_i \] 近似于： \[ L_o(p, \omega_o) \approx \dfrac{\int_{\Omega+} V(p,\omega_i) \ d\omega_i}{\int_{\Omega+} d\omega_i} \cdot \int_{\Omega+}L_i(p, \omega_i) \ f_r(p, \omega_i, \omega_o) \ cos\theta_i \ d\omega_i \] 仔细观察这个式子，其左边是 visibility，右边是 shading，并且两者可以相乘。

这就是 Shadow Mapping 最基础的思路！

那我们说什么时候上式是准确的呢？

当积分范围很小的时候是准确的！小到一定程度积分范围就变成一个点，那什么积分范围是一个点呢？即点光源和方向光源的时候，只有某一个方向或者只有一个点上有光照。

所以，对于点光源和方向光源，这样拆分没有问题！这就是 Shadow Mapping的数学支撑！

另一种情况是当积分函数 smooth 的时候是准确的！即对于 BRDF 是 Diffuse 和 面积光源上的 Radiance 是常量的时候，这样做也没有问题！而对于 Glossy 的 BRDF 是不太适合 Shadow Mapping 的。

4. PCSS

4.1 PCF

PCF(Percentage Closer Filtering) 一开始是用来做 shadows' edges 的抗锯齿，如果用在软阴影上叫 PCSS(Percentage closer soft shadows)。

注意，PCF 做的是阴影比较结果的 filter，不是对最后已经有锯齿的阴影进行 filter，也不是对 shadow map 进行 filter。

做 PCF 的时候，对于每个着色点，把它投影到光源之后，不是找它对应的一个像素，而是找它对应像素周围的一圈像素，对于这一圈像素的深度都和着色点的深度进行一个比较，然后把比较的值平均起来。这里 filter 的是什么？filter 的是任意一个 shading point 做的很多次不同的阴影的比较结果。

图7  PCF对阴影比较结果做一个filter

其实可以发现 filter 的大小很重要，当 filter 很小的时候，阴影比较硬，当 filter 很大的时候，阴影比较软。

4.2 PCSS

所以我们能否用 PCF 来获得软阴影？答案是可以的。那么问题来了，在将硬阴影变成软阴影时候，filter 大小应该多大呢？不同地方的 filter 都应该一样大吗？

图8  离遮挡物更近的地方阴影硬，离遮挡物更远的地方阴影软

观察上面的图，可以发现笔尖的部分阴影比较硬，笔杆的部分阴影比较软。换句话说，离遮挡物更近的地方阴影更硬，离遮挡物更远的地方阴影更软。

所以，如果我们想要做一个软阴影的效果，就应该在不同地方采用不同大小的 filter，而且 filter 的大小应该与遮挡物的距离（blocker distance）有关。

图9  PCSS示意图

上图很直观展示了思路，当 Blocker 距离 Receiver（Shading Point） 更远的时候，根据图中相似三角形，半影范围更大；当 Blocker 距离 Receiver 更近的时候，半影范围更小。即： \[ w_{Penumbra} = (d_{Reciver} - d_{Blocker}) \cdot w_{Light} / d_{Blocker} \] （补充：对于面光源来说，原本是不可能生成一个 Shadow Map 的，正常情况了为了模拟面光源的软阴影，用的是点光源的方式，把相机放在面光源中间然后看向场景）

PCSS 完整的算法流程：

  • Step1：Blocker Search

    计算 blocker 的平均深度（Shadow Map 周围一圈遮挡住的平均深度）

  • Step2：估计半影

    根据 blocker 平均深度决定 filter 大小

  • Step3：PCF！

有一个问题就是，计算 blocker 平均深度的时候，shadow map上周围一圈的大小应该取多大？

图10  如何做blocker search

上面的图给了一个很好的启发式方法，着色前与光源形成的视锥在 shadow map 上覆盖的范围可以来做 blocker search，对于没有遮挡的点不用管，对于遮挡的点（blocker）求其平均深度。

### 4.3 PCF背后的数学

对于过滤 / 卷积有如下公式，其中，\(\omega *f\) 表示对函数 \(f\) 的卷积操作 \(\omega\)，\(\mathcal N (p)\) 表示 \(p\) 的邻域内的点。
\[ [\omega * f](p) = \sum_{q \in \mathcal N (p)} \omega(p,q) f(q) \] 在 PCSS 中有如下公式，其中，\(\chi^+\) （读法 kai 四声）是符号函数，当变量大于 0 时函数值为 1，小于 0 时函数值为0。\(D_{SM}(q)\) 表示 Shadow Map 中像素点 \(q\) 存储的深度值，\(D_{scene}(x)\) 表示从光源到着色点 \(x\) 的深度值。 \[ V(x) = \sum_{q\in \mathcal N(p)}\omega(p,q) \cdot \chi^+[D_{SM}(q)-D_{scene}(x)] \]

图11  PCSS背后的数学原理

因此，PCF 不等于将 Shadow Map 做个 filter： \[ V(x) \ne \chi^+\{[\omega * D_{SM}](q)-D_{scene}(x)\} \] 同样，PCF 也不等于对最后生成的图像空间做个 filter： \[ V(x) \ne \sum_{y \in \mathcal N (x)} \omega(x, y) V(y) \]

5. VSSM

让我们重新回顾一下 PCSS 的三个步骤，可以发现第一个步骤（计算 blocker 的平均深度）和第三个步骤（计算 PCF）是比较慢的，原因就是这两个步骤都需要做这样一个操作：就是在整个一个区域内，要看区域里面所有的纹素记录的深度，并和某个深度相比。（补充：一个加速的方法是不完全遍历区域内所有的点，我们只采样部分点。但是这样结果会比较 noisy，最后还需要在图像空间下做个降噪。）

为了解决 PCSS 第一个步骤和第三个步骤速度慢的问题，就有人提出了 VSSM 算法（Variance Soft Shadow Mapping）。

回想一下第三个步骤的 PCF（percentage closer filtering） 算法，它做的事情就是把 Shadow Map 上记录的某一个区域内所有点的深度和着色点（shading point）的深度作比较，结果要么 1 要么 0，然后把结果平均起来。换句话说，不就是看这个区域内有多少百分比的点比着色点更近嘛 ~

所以重点就是如何快速求出这个百分比。假如给你一个直方图，你能很快就估计出这个百分比，那现在我们想大概估计一下百分比，甚至连直方图都不需要，就把它当作一个正态分布来估计百分比。

定义一个正态分布需要什么？均值和方差即可。

VSSM 提出的关键思路即是如此：快速地估计区域内的均值和方差。

  • 均值：用 MIPMAP （方形 Shadow Map） / Summed Area Tables（SAT，矩形 Shadow Map）

  • 方差：\(Var(X) = E(X^2) - E^2(X)\)，所以只需要再记录深度的平方即可！

好，得到了均值和方法后就可以得到正态分步，那么得到了正态分布后如何估计百分比？用 CDF ！

但是这个积分没有解析解，只有数值解，所以 VSSM 又做了另一件聪明的事 —— 切比雪夫不等式： \[ P(x \gt t) \le \dfrac{\sigma^2}{\sigma^2 + (t-\mu)^2} \ (for \ t \gt \mu) \]

图12  切比雪夫不等式

切比雪夫告诉我们一件事：如果我们知道了任何一个分布的均值和方差，就用上面公式的右项来估计CDF。

时间复杂度：范围内深度均值：O(1)、范围内平方深度均值：O(1)、切比雪夫不等式：O(1)。

好了，上面一直讲的是 PCF 第三步怎么加速，那么 PCF 第一步怎么加速呢？

还记得第一步求的是什么吗？是遮挡物的平均深度。即对于深度 \(z\) 小于着色点深度 \(t\) 的 blocker 的平均深度 \(avg. Z_{occ}\)，对于非遮挡物的平均深度，记作 \(avg. Z_{unocc}\)。所以我们有如下公式： \[ \dfrac{N_1}{N}z_{unocc} + \dfrac{N_2}{N}z_{occ} \] 其中，\(N\) 是区域内纹素总个数，\(N_1\) 是区域内非遮挡物纹素个数，\(N_2\) 是区域内遮挡物纹素个数。

\(N_1\) 和 \(N_2\) 如何求呢？ 可以观察到 \(\dfrac{N_1}{N}\) 就是指非遮挡物纹素所占的百分比呀！百分比怎么快速估计？切比雪夫不等式！即 \(N_1 / N = P(x \gt t)\) ！！

现在就剩下 \(z_{unocc}\) 不知道了，这里 VSSM 做了一个大胆的假设，即 \(z_{unocc} = t\) （阴影投射的地方是平面）。

SAT 即二维前缀和。每次计算区域内的和，如下图所示：

图13  Summed Area Tables

6. MSM

MSM（Moment Shadow Mapping）是针对 VSSM 的不足提出来的，由于 VSSM 使用了切比雪夫不等式，所以在深度分布不满足切比雪夫不等式的时候就会有问题。

VSSM 的限制：漏光、非平面的阴影接收物、当 \(t \gt z_{avg}\) 时切比雪夫不准。

MSM 的目标：更精确地去描述一个分布。

MSM 的想法：使用更高阶的矩来描述一个分布。

什么是矩？

  • \(x, x^2, x^3, x^4,...\) 记录多少次的方就是多少矩。
  • 所以，VSSM就是用了两阶矩（深度和平方深度）。

矩能做什么？

  • 结论：\(m\) 阶矩就能表示 \(m / 2\) 个阶梯的函数；
  • 通常，4 阶矩足够好去估计精确的深度的 CDF。

7. 距离场软阴影

距离场软阴影（Distance field soft shadows）。

什么是距离函数（Distance field / Distance function）呢？

距离函数定义了空间中的任何一个点到某个物体的表面的最小距离。物体外的点到物体表面距离为正，反之为负。

SDF 有一个非常不错的性质：可以表示物体的边界，如下图所示。（补充：SDF 背后的理论，最优传输 Optimal Transport）

图15  SDF

7.1 距离函数的用法

  • 用法一

    用 Ray Marching（Sphere Tracing）去做一个 光线-SDF 求交，

    隐含在其背后的思想是：SDF 的值表示不与物体相交的一个”安全“距离，

    因此，每个时间在 \(p\) 点向前走 \(SDF(p)\) 距离。

    图16  Ray marching(Sphere tracing)

  • 用法二

    使用 SDF 来生成软阴影，即光源大概有多少（Percentage）被挡住。

    隐含在其背后的思想是：SDF 的值表示从 eye 看到的“安全”角度。安全角度越小，visibility 越少。

    图17  用SDF确定“安全”角度

    现在假设 \(eye\) 从 \(o\) 点开始，朝着 \(light\) 中心方向做 \(sphere tracing\)，第一步走到 \(p_1\)，\(p_1\) 告诉我们走到 \(p_2\)，\(p_2\) 告诉我们走到 \(p_3\)，在这个过程中，在 \(p_1\) 的位置上可以得到一个圆，这时候从 \(o\) 点做圆的切线就得到了“安全”角度，最后的“安全”角度就是所有步得到的角度取最小值即可。

    图18  用SDF确定“安全”角度原理图

    如何计算每一步的夹角呢？

    确实可以用 arcsin 函数来计算，比如：\(\arcsin \dfrac{SDF(p)}{p - o}\)，但是在 shader 反三角函数太费时了，我们希望能避免，而 \(\dfrac{SDF(p)}{p-o}\) 其实就能反映夹角大小程度，所以我们使用下面公式： \[ min \left \{ \dfrac{k \cdot SDF(p)}{p-o}, 1.0 \right \} \] 其中，\(k\) 是一个常数，而且我们还可以发现 \(k\) 越大，夹角的变化范围也越小，即软阴影变化范围也越小 ，即 \(k\) 决定了半影（软阴影）的大小！

    图19  从上到下，k = 2, 8, 32.

  • SDF 的优缺点

    • 优点：快、高质量
    • 缺点：需要预计算、需要很多存储
