Frame 类

这里的 Frame 单词既不是指框架，也不是指一帧，而是指坐标系。

Frame 类用于存储一个三维的正交坐标系，其包含三个成员变量：s，t 和 n，它们分别代表了 x 轴，y 轴和 z 轴的向量。

它主要用于不同笛卡尔坐标系之间的变换和高效计算某些量（如 \(cos\theta\)，\(sin\theta\) 等）

sinPhi 的实现原理

当给定一个局部坐标系下的单位向量 \(v\)，sinPhi 函数的作用是返回 \(v\) 在球坐标系下的 \(\phi\) 参数的正弦值。

根据三维空间中球坐标系的定义，可以将向量 \(v\) 转换为球坐标系下的三个参数：\(\theta\) 为 \(v\) 与 \(z\) 轴的夹角，\(\phi\) 为 \(v\) 投影在 \(xy\) 平面上与 \(x\) 轴的夹角，\(r\) 为向量的模长。

直角坐标与球坐标转换的具体公式为： \[ \begin{align} &x=r\sin \theta \cos \varphi \\ &y=r\sin \theta \sin \varphi \\ &z=r\cos \theta \end{align} \]

接下来，需要计算 \(v\) 在球坐标系下 \(\phi\) 参数的正弦值，根据球坐标系的定义，可以将其表示为： \[ \sin\phi = \frac{y}{r\sin\theta} = \frac{y}{\sin\theta} \]

但是，在 \(\theta=0\) 的情况下，上式分母为零，无法计算。因此，在 sinPhi 函数中，需要先检查 \(\theta\) 是否为零，如果是，则直接返回 \(1\)：

否则，将 \(y\) 除以 \(r\sin(\theta)\) 得到 \(\sin(\phi)\) 的值，使用 clamp 函数将其限制在 \([-1,1]\) 的范围内并返回。

Integrator 类

Integrator 抽象类是渲染技术的基类。在 Nori 中，不同的渲染技术统称为 Integrator，因为它们在高维空间上进行积分。每个 Integrator 都代表了解决光传输方程的特定方法，通常在特定情况下表现优异，但同时也受到其自身固有局限性的影响。

Sampler 类

Sampler 抽象类是一个采样生成器，负责生成随机数流，并将其传递给一个实现了 Integrator 接口的类，当 Integrator 实现类在计算某条光路入射的 Radiance 时会用到。

最容易想到的采样生成器就是关于 MT（即 C++ std::mt19937 随机数生成器）随机数生成器的 wrapper，它在 independent.cpp 中被实现（这样命名是因为它生成了统计数据上独立的随机数）。

为了提升收敛，更好的采样生成器可以使用 分层法 或者 低差异序列（例如：Halton、Hammersley、Sobol point sets）。此类的另一种用法是故意生成相关的随机数，例如作为 Metropolis-Hastings 积分方法的一部分。

一个 sampler 和一个渲染算法之间通用的接口如下：在开始渲染一个像素前，渲染算法会调用 generate()，然后可以计算第一个像素样本，在此之后需要调用 advance()，重复这个过程直至计算完所有像素。当计算一个像素样本的时候，渲染算法使用 next1D() 和 next2D() 请求随机数。

从概念上看，正确的思考方式如下：对于像素中的每个样本，采样生成器生成无限维随机数超立方体中的一个（假设的）点。

渲染算法可以使用 next1D() 和 next2D() 函数请求该点的后续 1D 或 2D 组件。此类的高级实现对前 n 个分量相对于像素内采样的其他点的分层做出了某些保证。

Diffuse 类

Diffuse 类，即材质是 Diffuse 类型，或者说是 Lambertian BRDF 模型。

eval 函数计算漫反射，返回的是 \(\dfrac{albedo}{\pi}\)，其中 \(albedo\) 是表面的固有色，而这里要除以 \(\pi\)，目的是保持能量守恒。可参考 “为什么PBR中Lambert光照要除PI? - Jeffrey Zhuang的文章 - 知乎

pdf 函数计算采用的概率密度，由于采用重要性采样，所以返回 \(\dfrac{cos\theta}{\pi}\)。

sample 函数采用 cosine 重要性采样，使用了 Warp::squareToCosineHemisphere() 方法。

Mirror 类

Mirror 类，即材质是理想镜面 BRDF 类型。

eval 函数总是返回 0。

pdf 函数总是返回 0。

sample 函数返回入射方向的反射方向。

Dielectric 类

Dielectric 类，即材质是电介质类（比如玻璃）。

eval 函数总是返回 0。

pdf 函数总是返回 0。
