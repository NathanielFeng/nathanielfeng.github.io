oop

  • 封装

  • 继承

  • 多态：相同的消息请求，执行不同的代码体，有不同的行为后果。

    • 静态多态：根据对象的静态类型和参数表中的参数的静态类型确定目标代码体（模板和函数重载）
    • 动态多态：根据对象的动态类型和参数表中的参数的静态类型确定目标代码体（虚机制）

    封装可以使代码模块化，继承可以扩展已存在的代码，两者目的都是代码重用，多态是为了接口

  • 重载：函数名相同，函数的参数个数、参数类型或参数顺序三者中必须至少有一种不同。函数返回值的类型可以相同，也可以不相同。发生在一个类内部。

  • 重写：也叫覆盖，一般发生在子类和父类继承关系之间。子类重新定义父类中有相同名称和参数的虚函数。

  • 重定义：也叫隐藏，子类重新定义父类中有相同名称的非虚函数 ( 参数列表可以不同 ) ，指派生类的函数屏蔽了与其同名的基类函数。发生在继承中。

存储空间

静态内存：栈区、代码区、数据区（全局数据区、常量数据区）

动态内存：全局堆区

包含警戒

作用：同一个cpp文件中，可能会多次include同一个头文件，造成重复定义。加入包含警戒后，预编译后，各头文件在该cpp中只多出现一次。

包含警戒命名应该唯一。

单一变量原则

在一个编译单元中，标识符（变量、函数、类）只能定义一次，可以多次声明。

  1. 头文件中可放声明，以及类定义，但不应该放变量定义和函数定义；
  2. 函数定义及变量定义，只能在cpp文件中定义一次。

virtual

虚函数是指一个类中你希望重载的成员函数，当你用一个基类指针或引用指向一个继承类对象的时候，你调用一个虚函数，实际调用的是继承类的版本。

虚表实现多态的过程： 参考博客：C++虚函数表剖析、陈伟老师PPT、虚函数详解

Notes：

  1. 静态成员函数、构造函数、拷贝构造函数不能是虚的，析构函数可以是虚的；
  2. 若类中有其他虚构函数，那么析构函数也应该是虚的；

纯虚函数：为了实现一个接口，起到规范派生类的作用。纯虚函数一定没有定义，但它们必须要在派生类中重写；纯虚函数所在类称为抽象类，抽象类不能定义实例。但是可以定义它们的指针。

抽象类：具有纯虚函数的类为抽象类，除此之外就有类的其他特性，比如也能具有普通函数和成员变量，一个类只能继承一个抽象类；

接口：类中只有函数，且都为纯虚函数。接口是特殊的抽象类，一个类可以继承多个接口。

引用和指针的区别

  1. 指针是一个变量，只不过这个变量存储的是一个地址，指向内存的一个存储单元；而引用仅是个别名；

  2. 引用使用时无需解引用(*)，指针需要解引用；

  3. 引用只能在定义时被初始化一次，之后不可变；指针可变；

  4. 引用不能为空，指针可以为空；

  5. 指针占用空间，引用不是新定义一个变量，它只是表示该引用是目标变量名的一个别名，它本身不是一种数据类型，因此引用不占用存储单元。

struct和class的区别

  1. class中默认的访问权限是private、继承默认是private继承；struct中默认的访问权限是public，继承默认是private继承。

new和malloc的区别

  1. malloc与free是C++/C语言的标准库函数，new/delete是C++的运算符。

  2. new的功能是在堆区新建一个对象，并返回该对象的指针。新建对象的意思就是，将调用该类的构造函数，因为如果不构造的话，就不能称之为一个对象。而malloc只是机械的分配一块内存，如果用mallco在堆区创建一个对象的话，是不会调用构造函数的。严格说来用malloc不能算是新建了一个对象，只能说是分配了一块与该类对象匹配的内存而已，然后强行把它解释为这是一个对象，按这个逻辑来，也不存在构造函数什么事。同样的，用delete去释放一个堆区的对象，会调用该对象的析构函数。用free去释放一个堆区的对象，不会调用该对象的析构函数。

实例变（常）量：每个对象独立拥有的数据成员

实例方法：访问实例变（常）量的成员函数

类变量：类具有的属性或应由类中所有对象共享的属性

  • C++中用静态数据成员表示
  • 类变（常）量的内存位置
  • 类变（常）量的初始化

类方法：类属的行为表示

  • C++中用静态成员函数表示
  • 不能访问任何非静态成员（数据成员和函数成员）
  • 没有this指针，也不能带const修饰

const关键字总结

1. const和指针

指向常量的普通指针：

const T* p [=exp];

const int* p = &a; int const* p = &a;两种形式等价，意思是：p一个指向整形变量的指针，且不能通过p来修改a的的值。像int* p4 = p1;是错误的，因为如果成立就能通过p4来修改a的值，这和我们的初衷违背。

指向变量的常指针：

T* const p = exp;(必须初始化)

指向常量的常指针：

const T* const p = exp;(必须初始化)

关于字符串的说明：

char* 和 char[]定义字符串：

  • char* str = "This is a string!"; 等价于const char* str = "This is a string!"（是为了照顾C语言的兼容性），不允许通过str修改字符串，此时字符串放在常量数据区；

  • char str[] = "This is a string!"; 可以通过str修改字符串，此时字符串放在变量数据区；

2. const和引用

const T& p = val1表示不能通过p来修改val1，如p = val2是错误的。此外，常量的引用应该加上const，如：const int p = 1否则报错。

3. 函数返回引用

函数返回值是引用类型时，该返回值可以作为左值使用。不能返回局部变量的引用，这是因为局部变量存储在栈区，当函数调用结束时会释放栈空间，导致引用的是垃圾值。

static关键字用法总结

1. 静态局部变量

（1）该变量在全局数据区分配内存(局部变量在栈区分配内存); （2）静态局部变量在程序执行到该对象的声明处时被首次初始化，即以后的函数调用不再进行初始化(局部变量每次函数调用都会被初始化); （3）静态局部变量一般在声明处初始化，如果没有显式初始化，会被程序自动初始化为0(局部变量不会被初始化); （4）它始终驻留在全局数据区，直到程序运行结束。但其作用域为局部作用域，也就是不能在函数体外面使用它(局部变量在栈区，在函数结束后立即释放内存);

2. 静态函数/全局变量

（1）静态函数/全局变量不能被其它文件所用(普通函数/全局变量可以); （2）其它文件中可以定义相同名字的函数/变量，不会发生冲突(自然了，因为static隔离了文件，其它文件使用相同的名字的函数/变量，也跟它没关系了);

3. 静态数据成员

类变量，存储在全局数据区。

4. 静态成员函数

静态成员函数不能访问非静态(包括成员函数和数据成员)，但是非静态可以访问静态

拷贝构造函数

  • 从无到有地创建一个新对象，是一种特殊的构造
  • 不同于普通构造函数，参数必须且只需是一个本类对象

1. 什么时候会用到拷贝构造函数？

  • 对象以值传递的方式传入函数参数
  • 对象以值传递的方式从函数返回
  • 对象需要通过另外一个对象进行初始化

2. 浅拷贝和深拷贝

2.1 浅拷贝

缺省的拷贝构造函数就是访问控制为public的浅拷贝。

针对当前对象的存储区域进行拷贝，即拷贝实例变量。

浅拷贝会出现的问题：

  • 不能处理静态数据成员；
  • 不能处理动态成员；

运行会发现IDE会报出”引发异常“的中断信息。

分析这段程序，当main函数执行完毕时会将栈区清空，也就是说，会依次调用b2、b1的析构函数。

当执行完b2的析构函数后，b2指向的堆区对象空间被释放，再调用b1析构函数时，会因为运行时“访问非法数据”而报错引发异常。

2.2 深拷贝

自定义拷贝构造函数

B(const B& b) { pA = new A(*b.pA); }

3. 什么时候用到自定义拷贝构造函数？

  • 深拷贝

  • 禁止拷贝 & 防止按值传递对象

总结一下：若要完全禁止拷贝，最好自定义一个private且没有实现的拷贝构造函数；或在C++1z中，用My(const My& m) = delete;

4. 关于拷贝构造函数注意点

  • 拷贝构造函数必须以引用的方式传递参数。这是因为，在值传递的方式传递给一个函数的时候，会调用拷贝构造函数生成函数的实参。如果拷贝构造函数的参数仍然是以值的方式，就会无限循环的调用下去，直到函数的栈溢出。
