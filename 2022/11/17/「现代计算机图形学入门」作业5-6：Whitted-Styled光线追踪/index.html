「现代计算机图形学入门」Whitted-Styles光线追踪

1. 代码框架

整个代码框架可以分为 6 大类：

  • 工具类：global.hpp、vector.hpp
  • 光源类：Light.hpp
  • 物体类：Object.hpp、Sphere.hpp、Triangle.hpp
  • 场景类：Scene.hpp（Scene.cpp 基本没啥代码不考虑）
  • 主函数：main.cpp
  • 渲染类：Renderer.hpp（Renderer.cpp 同上）

2. 工具类 & 光源类

2.1 global.hpp

global.hpp 中主要定义了一些全局变量和函数，如：\(\pi\) 值、材质类型、求随机数、一元二次求解函数等等；

2.2 Vector.hpp

Vector.hpp 中主要提供了常见的向量操作，如：向量点乘、叉乘、单位化、插值等。

2.3 Light.hpp

Light.hpp 中定义了光源类，包括光照强度和光源位置。

3. 物体类

3.1 Object.hpp

定义了物体的材质类型（materialType）、折射率（ior）、漫反射系数（Kd）、漫反射颜色（diffuseColor）、高光系数（Ks）、高光衰减指数（Exponent），

以及三个虚函数：判断是否和光线相交的 intersect、获取表面属性的 getSurfaceProperties、计算漫反射颜色的 evalDiffuseColor。

3.2 Sphere.hpp

主要重载了 intersect 判断光线是否和球面相交，并将离光线起始点最近的交点 （tNear）返回。通过将光线方程代入球体方程即可得到；

还重载了 getSurfaceProperties 来获取球面交点处的法向量。

3.3 Triangle.hpp

这里的三角形在场景中对应着地面，地面由两个三角形拼接而成。

除了基类 Object 的成员变量，还定义了三角形数量（numTriangles）、顶点数组（vertices）、顶点下标数组（vertexIndex）和 uv 坐标数组（stCoordinates）

后三者全使用智能指针 unique_ptr 来管理是防止内存泄露，即使异常发生也可帮助避免资源泄露，而无需手动 delete，这在三角形面数很多的场景中更加有用。

此外，重载了 intersect 判断光线是否和所有的三角形相交，并将相交的第几个三角形（index）、最近的交点 （tNear）以及 重心坐标（u，v）返回。

还重载了 getSurfaceProperties 来获取三角形交点处的法线和经过重心坐标插值后的 uv 坐标。

最后还重载了 evalDiffuseColor 自动生成一张纹理图（即地板）。

4. 场景类

4.1 Scene.hpp

主要定义了屏幕的宽高、摄像机的 fov 参数、光线最大递归深度、epsilon（后面会讲作用）、场景背景颜色；

还定义了两个数组，分别存放场景中的所有物体和所有光源。

5. 主函数

main.cpp

实例化了场景和所有的物体、光源。需要注意的是如果观察物体的坐标值，可以推测默认是在右手坐标系下，

摄像机看向 -Z 轴。

6. 渲染类

6.1 Renderer.cpp

终于说到最重要的部分渲染这一块了，这是最精髓的地方，我们按顺序一点点看。

6.1.1 反射向量

要求一个向量 \(I\) 经过反射后的反射向量 \(R\)。

推导很简单：

由于入射角等于出射角，向量 \(I\) 和 \(R\) 可以形成一个菱形，

故 \(I\) 的模在法线 \(N\) 上的投影的两倍就是菱形的对角线，即：\(2*(I·N)*N\)。

所以反射向量 \(R=I-2*(I·N)*N\)。

6.1.2 折射向量

我们先看折射向量怎么算，这里计算折射向量用到了斯涅尔定律（Snell's law），即 入射介质折射率 \(\eta_{i}\) 与 入射角 \(sin \theta_{i}\) 之积等于出射介质折射率 \(\eta_{t}\) 与 出射角 \(sin \theta_{t}\) 之积。

根据上图，我们很容易求出 \(sin \theta_{t}\) 和 \(cos \theta_{t}\)，它们正是我们要求的折射向量在水平方向和竖直方向的投影长度。

剩下要做的就是如何用已知条件：入射向量 \(I\)、法线 \(N\)、入射角 \(\theta_{i}\) 表示水平方向的向量和竖直方向的向量。

竖直方向很简单就是 \(-N\)，水平方向是 \(\dfrac{(I·N)*N - I}{sin\theta_{i}}\)（都需要单位化），所以： \[ refract = sin\theta_{t} * \dfrac{(I·N)*N - I}{sin\theta_{i}} + cost\theta_{t} * (-N) \] 化简一下可得：\(refract = -\dfrac{\eta_{i}}{\eta_{t}} * I + \left [ \dfrac{\eta_{i}}{\eta_{t}} * (I·N) - cost\theta_{t} \right ] * N\)。

很明显，此公式代码 eta * I 不一样，多了个负号，这是因为代码中的入射向量与我们推导的相反。

此外，我们还需要考虑光线在物体内和物体外情况。在物体外时，物体表面法线与光线的夹角大于90°，余弦值为负，由于计算的公式与光线的方向相反，所以我们需要把余弦值取反；在物体内时，物体表面法线与光线的夹角小于90°，本来是需要将余弦值取反的，但是由于法线正好也是反的，所以余弦值不需要取反了，只需要将法线同时，同时交换下折射介质。

6.1.3 菲涅尔方程

菲涅尔方程求的是：入射光的角度与反射光的强度、折射光的强度之间的关系。这里直接根据老师给出的公式即可。

不过让我存在疑惑的是代码中的 Rs 和 Rp 好像写反了，不过不影响就是就是了。

6.1.4 光线与物体求交(trace)

这一步将光线与所有物体求交，并返回距离光线起点最近的点的信息。参数 orig 和 dir 描述了光线起点的方向，objects 是求交的物理列表。payload是一个结构体，用于存储交点的信息，如相交的物体指针（hit_obj），离光线起点的距离（tNear），相交三角形的下标（index，如果相交的是三角形），uv坐标（uv）。

6.1.5 Whitted风格的光线追踪算法(castRay)

castRay 实现的是一个递归的计算光线与物体交点处颜色的过程，规定最大深度是 5。首先通过计算光线与所有物体的交获得交点处的uv坐标和法线，用于后续的着色。

然后根据材质的不同，分别进行着色：

如果是 REFLECTION_AND_REFRACTION，又存在反射和折射，则分别计算反射光线和折射光线，分别继续求反射光线和折射光线与场景物体的交点，然后根据菲涅尔定理混合其颜色。

需要注意的是我们计算向量起点使用 epsilon 做了一些调整。这是由于计算机精度，直接使用起点产生噪声值，如下图：

这是因为相交的由于精度损失，跑到了内部，所以我们需要使用一个 epsilon将其救出来。

对于反射光线和折射光线，它们可能与法线同向也可能异向，所以我们通过法线与光线的点乘来判断。当同向时，按照法线方向移动一点点，当异向时，按照法线的相反方向移动一点点。

如果是 REFLECTION，即只存在反射，只用考虑反射即可，不需要考虑折射。

对于漫反射材质，我们依据Phong光照模型计算其着色。

我们先判断交点是否在阴影里，即是否能被光线看到，做法和shadow mapping很像，判断从交点向光源发出光线打到第一个点的距离和光源到交点的距离是否相等即可。在阴影里，则漫反射光为 0，否则按照光照模型计算。
